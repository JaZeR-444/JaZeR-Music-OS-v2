# Phase 3 - Deferred Dashboard Items

## Purpose

This log prevents deferred ideas from quietly reappearing on the Master Dashboard during manual build.

## Deferred Or Cut Items

| Item | Decision | Reason | Revisit trigger |
|---|---|---|---|
| Recent Sessions | Cut | Historical context, not action context. | Do not revisit for dashboard. Use Sessions inside Song pages if needed. |
| Weekly Review template | Defer | Must wait for at least 14 days of real use. | After 14+ days, build inside Song Lab, not dashboard. |
| Due This Week as dashboard zone | Defer | `Focus Now` is the primary dashboard task view. | Revisit if focused work repeatedly misses real deadlines. |
| Release Pipeline zone | Defer build | Releases DB is not built until a real release is about 60 days out. | Build when Releases DB is activated. |
| Open Tasks Count filter on Songs in Production | Defer | `Open Tasks Count` rollup is deferred. | Add after Day 30+ or real task volume. |
| Last Session / Stale Warning dashboard signals | Defer | Stale formulas need real session history. | Day 30+ after session habit exists. |
| Weekly Reset button | Defer | Same as Weekly Review; too early at launch. | After 14+ days of real use. |
| Brand link section | Cut | Brand Reference Page owns static references. | Revisit only if active brand workflow volume appears. |
| Archive / Vault previews | Cut | Dashboard should not surface archive content. | Never for Master Dashboard. |
| All Tasks view | Cut | Too broad for a focus surface. | Use Tasks DB or Launch Hub. |
| All Songs view | Cut | Catalog view belongs in Song Lab. | Use Songs DB or Song Lab. |

## Allowed Future Dashboard Additions

Allowed only after explicit approval:

- A compact link to the latest Weekly Review page after the Weekly Review system exists.
- A Release Pipeline toggle after Releases DB exists.
- A stale-song warning after Day 30+ and after `Last Session` / `Days Stale` exist.

Any future addition must keep the dashboard at five zones maximum.
